# Teachable Mastra Admin

Mastra project scaffold (`create-mastra`) hosting the **Teachable Admin Assistant** agent. Every OpenAPI operation in `open-api/admin-api.yaml` is exposed as a Mastra tool (67 tools), generated by `@teachable/admin-tools-codegen`.

## Local setup

1. Build the HTTP client package (the Mastra app imports it via a monorepo-relative path so `mastra build` can bundle it):

   ```bash
   cd packages/teachable-admin-http-client && npm run build
   ```

2. Generate tools into `src/generated/` (also runs automatically before `mastra build`):

   ```bash
   cd apps/teachable-mastra-admin && npm run codegen:admin-api
   ```

3. In this directory, install dependencies and copy environment variables:

   ```bash
   cd apps/teachable-mastra-admin
   npm install
   cp .env.example .env
   ```

4. Edit `.env`: set `OPENAI_API_KEY`, `TEACHABLE_ADMIN_BASE_URL`, and `TEACHABLE_ADMIN_CONSUMER_CUSTOM_ID` to non-production test values (never commit real secrets).

5. Run the Mastra dev server:

   ```bash
   npm run dev
   ```

## Scripts

| Script                 | Purpose                                      |
| ---------------------- | -------------------------------------------- |
| `npm run codegen:admin-api` | Regenerate `admin-api-tools.ts` from YAML |
| `npm run test`         | Registry parity tests vs OpenAPI model       |
| `npm run typecheck`    | TypeScript `--noEmit`                        |
| `npm run lint`         | ESLint                                       |
